## Auto-discover Retro Channel

### Auto-discover retro channel when bot is added to a channel

Replaced the hardcoded `RETRO_CHANNEL_ID` env var dependency with automatic channel detection. When a user invites RetroRun to a channel (`/invite @RetroRun`), the `member_joined_channel` event fires, and the app automatically sets that channel as the retro target. The env var remains as a fallback for startup.

Created `listeners/channel-store.js` as a shared in-memory state module (`getRetroChannel`, `setRetroChannel`, `getBotUserId`, `setBotUserId`). Updated `app.js` to call `auth.test()` on startup to discover the bot's own user ID. Updated `app-home-opened.js` and `retro-submit.js` to use `getRetroChannel()`. Updated App Home limited view to guide users to `/invite @RetroRun`. Updated all tests to mock channel-store instead of env vars. Used TDD throughout: 7 new tests written first.

- **Sources:**
  - [Slack member_joined_channel event](https://api.slack.com/events/member_joined_channel)
  - [Slack auth.test API](https://api.slack.com/methods/auth.test)
