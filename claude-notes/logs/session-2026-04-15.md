### Create demo project channels with messages and canvases

Created `scripts/seed-projects.js` that programmatically sets up 3 project channels (#proj-aurora, #proj-beacon, #proj-cascade). Each channel gets a topic, 8 realistic team messages, and a project hub canvas with overview, team, architecture, sprint goals, and decision log. Added `channels:manage` and `channels:join` scopes to manifest for channel creation.

- **Sources:**
  - Slack Web API: `conversations.create`, `conversations.setTopic`, `chat.postMessage`, `conversations.canvases.create`

### Teardown, seed-members, and seed-retro-submissions scripts

Added 3 more scripts: `teardown.js` (clears messages and canvases from seeded channels), `seed-members.js` (joins bot to all project channels), and `seed-retro-submissions.js` (exercises the date-grouping canvas write flow sequentially).

Initially, teardown archived channels via `conversations.archive`. This created a catch-22: bots can't join archived channels (`is_archived` error) and can't unarchive without being in them (`not_in_channel` error). Revised teardown to clear channel content instead of archiving — channels stay active and reusable across seed runs.

> "Can't join archived channels and can't unarchive without being in them — classic Slack catch-22 for bots. These need to be unarchived manually in Slack or via an admin/user token."

- **Sources:**
  - Slack Web API: `conversations.archive`, `conversations.join`, `conversations.unarchive`
  - Slack Web API: `conversations.history`, `chat.delete`, `canvases.delete`
