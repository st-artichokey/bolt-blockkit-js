## Bug Fixes

### Add missing im:write scope for DM copies

The "Send me a copy" checkbox in the retro modal triggers a DM via `chat.postMessage` to the user's Messages tab. This requires the `im:write` OAuth scope, which was missing from `manifest.json`. Without it, DM copy requests silently failed.

- Added `im:write` to `manifest.json` bot scopes
- Added TDD test in `tests/manifest.spec.js` asserting the scope is present (red-green)
- Full suite: 62 tests passing

- **Sources:**
  - [Slack chat.postMessage docs](https://api.slack.com/methods/chat.postMessage)
  - [Slack OAuth scopes](https://api.slack.com/scopes)
  - Session log from 2026-03-30 (identified the missing scope)

### Replace broken deep link with native channel mention

The retro channel link in App Home used a `slack://channel?team=&id=...` deep link that didn't work in practice. Replaced it with Slack's native mrkdwn channel mention format (`<#CXXXXX>`), which Slack automatically renders as a clickable channel link. Manual testing confirmed the native mention works correctly.

- Replaced deep link with `<#${channelId}>` in `listeners/events/app-home-opened.js:76`
- Updated test to assert native channel mention format is used
- Full suite: 63 tests passing

- **Sources:**
  - [Slack mrkdwn reference](https://api.slack.com/reference/surfaces/formatting#mentioning-channels)
  - Manual testing in Slack App Home
